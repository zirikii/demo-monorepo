# Build Optus Demo — Full Marketing Site, Auth, and My Optus Dashboard

> Generated by the `mock-company-repo` skill (generate-and-build mode) and then
> implemented in this repo. Kept as the spec/record of what was built.

## Mission

Scaffold a production-quality **Optus UI clone** for demo purposes that looks and
feels like the real product at `https://www.optus.com.au/`. This is a UI/UX fidelity
demo, not a production app: dummy data, mock auth, local env vars, JSON persistence.
No real external services.

**Deliverable:** a single PR against `main` with a complete, lint-clean, build-passing
app under `apps/optus/`, wired into the pnpm workspace, runnable via `pnpm dev:optus`.

## Company profile

- **Product:** Australian telco — mobile plans (5G), nbn®/5G home internet, phones,
  prepaid, Optus Sport entertainment, and the My Optus self-service account.
- **Primary users:** Australian consumers managing mobile/internet services.
- **Core surfaces:** marketing home, mobile plans, home internet, phones, prepaid,
  Optus Sport, deals, store finder; My Optus overview/usage/plans/recharge/billing/settings.
- **Dummy-data theme:** Australian addresses, AUD pricing, en-AU locale, AEST fixtures.

## Repo context

- Monorepo (pnpm workspace). Scaffold under `apps/optus/` and wire into
  `pnpm-workspace.yaml` (via `apps/*`) and root `package.json` scripts.
- Model closely on the existing `apps/spark` telco demo for structure and conventions.

## Tech stack (matches other Next apps in the monorepo)

Next.js 15 App Router · React 19 · TypeScript (strict) · Tailwind v3 · Radix UI
primitives · `@tanstack/react-table` · `react-hook-form` + `zod` · `jose` mock auth ·
local JSON persistence · Vitest + Testing Library · Playwright walkthrough recorder.

Explicitly DO NOT integrate real databases, cloud providers, CMS APIs, OAuth, or
payment providers.

## Brand assets

- Palette: teal `#39A8AF` (primary/wordmark), yellow `#FECD03` (accent), ink `#1A1A1A`.
- Self-host logo/favicon/backdrops under `public/brand/` with a `brand-assets.json`
  manifest and `scripts/fetch-brand-assets.sh`.
- `optus.com.au` is blocked by the cloud network allowlist, so ship original SVG
  assets built from the documented palette and document the fallback; the fetch
  script pulls the official assets when run on an allowed network.

## Design system

Rounded, friendly geometric feel (Nunito Sans substitute). Teal primary buttons/links,
yellow accent blocks and highlights, generous rounded corners (`lg 12px`). Reusable
primitives: Button (cva variants), Badge, Accordion, Switch, EmptyState, Skeleton,
SiteHeader/SiteFooter, PageHero, PlanCard, PhoneCard, StoreFinder, BillsTable,
TopUpForm, MyOptusNav, SettingsTabs.

## Pages

- Marketing: `/`, `/mobile-plans`, `/broadband`, `/phones`, `/prepaid`,
  `/optus-sport`, `/deals`, `/stores`, `/privacy`, `/login`, `/signup`.
- My Optus (middleware-protected `(myoptus)` group): `/dashboard`, `/usage`,
  `/plans`, `/top-up`, `/bills`, `/settings` (+ `account`, `profile`, `team`,
  `integrations`).
- API route handlers wired to JSON: auth (login/logout/signup), `bills`, `top-up`,
  `mobile-plans`, `settings/team`, `settings/integrations`.

## Auth (mock)

`middleware.ts` protects the My Optus prefixes; unauthenticated → `/login?redirect=…`.
Session in a signed HTTP-only cookie (`jose` HS256, `DEMO_AUTH_SECRET`, fallback
`change-me`). Any credentials accepted. Logout clears the cookie.

## Data

Realistic dummy data under `data/` (mobile-plans, prepaid, broadband, phones, stores,
bills, top-ups, settings, sport, users). Route handlers read/write via an atomic
JSON store (temp file + rename).

## Tests

Vitest unit tests (formatters, session encode/decode + tamper), component tests
(PlanCard, MyOptusNav aria-current), and a mandatory route test per API handler
(GET filtering + POST/PATCH/DELETE mutations). Plus a Playwright walkthrough that
records a guided demo video.

## Quality gates

`pnpm lint`, `pnpm --filter optus-telco-demo typecheck`, `pnpm build:optus`, and
`pnpm --filter optus-telco-demo test` all pass. Accessible (aria-current on active
nav, aria-pressed on toggles, focus-visible rings, labelled icon buttons), responsive,
no console errors on happy paths.

# Changi Airport — passenger website demo (unofficial)

A pure React single-page app that recreates the look and feel of the Changi Airport
passenger website (`changiairport.com`) for demo purposes. It is part of the
`demo-monorepo` pnpm workspace and depends on the shared `@demo/ui` package.

> **Unofficial demo.** This project is not affiliated with, endorsed by, or connected to
> Changi Airport Group. Brand assets (the globe emblem, favicon and social icons) are
> sourced from the public Changi website purely for visual fidelity. All copy is original
> and written for demonstration only — flight data is mocked and deterministic.

## Stack

- **Vite 6 + React 19** (SPA), **TypeScript** (strict)
- **Tailwind CSS v4** — CSS-first `@theme` tokens in `src/index.css`
- **react-router-dom v7** for routing
- **lucide-react** icons, self-hosted **@fontsource** fonts (Figtree + Inter)
- **Vitest + React Testing Library** for unit tests
- **Playwright** for the recorded walkthrough video
- `@demo/ui` for the shared `cn` class merger and `<DemoRibbon>`

## Getting started

From the repo root:

```bash
pnpm install
pnpm dev:changi        # http://localhost:5176
```

Or inside this app:

```bash
cd apps/changi
pnpm dev               # dev server on :5176
pnpm build             # type-check + production build
pnpm lint              # eslint
pnpm typecheck         # tsc --noEmit
pnpm test              # vitest run
pnpm video             # record the Playwright walkthrough (needs a running dev server)
pnpm fetch-brand       # re-download official brand assets into public/brand
```

## Recording the walkthrough video

The walkthrough drives the whole site with a visible fake cursor and click ripples, then
saves an `.mp4` under `test-results/`.

```bash
pnpm dev &                                   # start the dev server on :5176
PLAYWRIGHT_BASE_URL=http://localhost:5176 pnpm video
# video is written to test-results/<spec>/video.webm (converted to mp4 by CI)
```

Chromium is required: `pnpm exec playwright install chromium`.

## How the mock data works

There is **no backend**. All content lives in typed modules under `src/data`. The flight
board is generated by a small seeded PRNG (`src/lib/flights.ts`) so the arrivals and
departures tables always show 25+ believable, stable rows — ideal for tests and a
flake-free walkthrough. Forms (newsletter, Changi Rewards join) are client-side only and
surface a success `status` message.

## Brand assets

Official assets were downloaded from `changiairport.com` and committed under
`public/brand/` (see `brand-assets.json` for source URLs and fetch date). The header
renders the "CHANGI airport singapore" lockup: the official multi-colour globe emblem
beside a taupe wordmark. Nothing is hotlinked at runtime — everything is self-hosted.

## Known limitations

- Flight data, promotions and rewards are illustrative and not live.
- "Login" / "Sign Up" are visual only — there is no real authentication.
- Maps, videos and photography are represented with brand-gradient panels rather than
  licensed imagery.
